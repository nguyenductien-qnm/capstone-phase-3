#Nội dung đề xuấtYêu cầu CDO thực hiện
1.1 Tái sử dụng cụm valkey-cart:6379 làm cache cho AI Review Summary. Cache key: reviews:summary:{product_id} — TTL động 4h→7d dựa trên lượng review (ADR-001). Subnet: Phối hợp CDO đặt cụm Valkey cache hoàn toàn trong Private Subnet để cách ly và đảm bảo an toàn bảo mật. Xác nhận cho phép AI ghi thêm key prefix reviews:* lên cụm Valkey hiện tại.
Bảo mật cụm cache: CDO xác nhận cấu hình cụm cache trong Private Subnet.CDO
1.2 Eviction Policy: volatile-lru (ADR-003 Option 1) Chỉ key có TTL (AI cache) mới bị evict. Key giỏ hàng (không TTL) được bảo vệ tuyệt đối. CDO sửa config Valkey thành volatile-lru.
CDO sửa code ValkeyCartStore.cs: bỏ dòng KeyExpireAsync cho giỏ hàng.CDO
1.3 CronJob dọn giỏ hàng cũ (chạy 2h sáng hằng ngày). AI tự viết script SCAN + xóa giỏ hàng idle > 30 ngày — CDO chỉ cần cho phép. Phê duyệt CronJob K8s chạy trên namespace chung.CDO
② Tích hợp dịch vụ Product Reviews (Contract) Chốt biến ENV Ref: contracts/product-reviews-integration.md
#Nội dung đề xuấtYêu cầu CDO thực hiện
2.1 Biến môi trường cho pod product-reviews: VALKEY_HOST, VALKEY_PORT, FLAGD_HOST, FLAGD_PORT, LLM_BASE_URL, LLM_MODEL Cập nhật Helm values / K8s ConfigMap để inject các biến ENV trên vào pod product-reviews.CDO
2.2 Resource Limits: CPU 100m→500m, Mem 128Mi→512Mi Đảm bảo pod không chiếm quá nhiều tài nguyên trên EKS. Xác nhận resource quota đủ cho giới hạn trên.CDO
2.3 Feature Flags trên flagd: llmReviewsCacheEnabled (bật/tắt cache) và llmRateLimitError (ép lỗi test). Thêm 2 flag vào cấu hình flagd hiện tại.CDO
③ Tích hợp dịch vụ Shopping Copilot [Extend] Extend Cần CDO phê duyệt Ref: contracts/shopping-copilot-integration.md
#Nội dung đề xuấtYêu cầu CDO thực hiện
3.1 Pod mới: shopping-copilot — Python gRPC trên cổng :50051 Resource: CPU 200m→1000m, Mem 256Mi→1024Mi. Dịch vụ Extend — AI tự build Docker image. Cấp namespace slot để deploy pod mới.
Cập nhật Helm chart thêm Deployment shopping-copilot.CDO
3.2 Envoy Proxy routing: thêm cluster + route cho gRPC-Web Prefix: /oteldemo.ShoppingCopilotService/ → cluster shopping-copilot:50051. Cập nhật cấu hình Envoy (frontend-proxy):
— Thêm cluster shopping-copilot
— Bật gRPC-Web filter
— Route prefix matchingCDO
3.3 Biến ENV để Copilot kết nối các dịch vụ hạ nguồn: PRODUCT_CATALOG_ADDR, CART_ADDR, PRODUCT_REVIEWS_ADDR, LLM_BASE_URL, FLAGD_HOST Inject biến ENV vào pod shopping-copilot qua Helm values.CDO
3.4 File Protobuf mới: shopping_copilot.proto AI đã tạo sẵn file proto trong pb/shopping_copilot.proto. CDO cần đồng bộ vào build pipeline của Frontend. Đồng bộ file proto vào các service consumer (Next.js Frontend) để generate client stubs.CDO

#Nội dung đề xuấtYêu cầu CDO thực hiện
4.1 IRSA (IAM Roles for Service Accounts) cho 2 pod AI: product-reviews và shopping-copilot cần quyền bedrock:InvokeModel trong vùng us-east-1. Tạo IAM Role với policy bedrock:InvokeModel trên các model:
— amazon.nova-lite-v1:0
— amazon.nova-micro-v1:0
— amazon.nova-pro-v1:0
Gắn IRSA vào ServiceAccount K8s tương ứng.CDO
4.2 Không hard-code credentials — chỉ dùng IRSA/SDK default chain. Tuân thủ security checklist: không lưu Access Key trong ENV hoặc ConfigMap. Xác nhận IRSA flow hoạt động trên EKS cluster hiện tại.CDO
⑤ Telemetry & Observability Contract Chốt spec Ref: contracts/*-integration.md §4
#Nội dung đề xuấtYêu cầu CDO thực hiện
5.1 Prometheus Metrics — AI cam kết export 7 custom metrics: llm_request_latency_seconds, llm_token_consumption_total, llm_error_rate_total, reviews_cache_hit_ratio, copilot_chat_requests_total, copilot_tool_calls_total, copilot_chat_latency_seconds Đảm bảo scrape target trong Prometheus config bao gồm 2 pod AI mới.
Cấp Dashboard panel trên Grafana cho AI team.CDO
5.2 Trace Context (Jaeger): W3C Trace Propagation không đứt đoạn. AI cam kết gắn gen_ai.system, gen_ai.model_id, gen_ai.usage.* lên mọi span gọi LLM. Xác nhận OTel Collector đang forward trace từ namespace chạy AI pods về Jaeger.CDO
5.3 Log Pipeline: stdout container → OpenSearch. AI cần log thô để chạy Drain3 Log Clustering (ADR-007 [Extend]). Xác nhận Fluentd/FluentBit đang ship log từ AI pods về OpenSearch index.CDO
⑥ Confirmation Gate — Phối hợp Frontend (ADR-006) Extend Tuần 3+ Không khẩn cấp — PoC đã hoạt động trên Streamlit
#Nội dung đề xuấtYêu cầu CDO thực hiện
6.1 Giao thức JSON Confirmation: Khi Copilot muốn ghi giỏ hàng, nó trả về JSON payload chờ xác nhận thay vì tự gọi API. Frontend cần render nút "✅ Xác nhận thêm vào giỏ" để user bấm. Chống Excessive Agency (OWASP LLM06). Lên kế hoạch sprint để Frontend team thêm component Confirmation UI vào trang Chat trên Next.js.ExtendCDO Frontend
6.2 Tool Tier phân loại: Tier 1 (Read-only): tự động. Tier 2 (Write): chờ xác nhận. Tier 3 (Critical): chặn tuyệt đối. Review & xác nhận bảng phân loại Tool Tier trong ADR-006 — đặc biệt là danh sách tool bị chặn (empty_cart, place_order, ship_order).CDO
✅ Cam kết của AI Team: Mọi thay đổi đều có Rollback Plan (tắt feature flag hoặc xóa pod). Không yêu cầu phát sinh thêm chi phí hạ tầng ($0). Code AI hoạt động độc lập — nếu AI pod sập, hệ thống gốc không bị ảnh hưởng.
